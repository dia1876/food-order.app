# Food Order App

飲食店向けの簡易オーダー管理アプリです。  
店舗業務の効率化を目的として、React + TypeScript を用いて個人開発しました。

## 概要
- 飲食店での注文受付を想定したWebアプリ
- 実運用を意識したUI設計とデータ管理
- フロントエンドからバックエンド連携まで一通り実装

## 開発背景
アルバイト先の飲食店で、
- 注文管理がアナログ
- 現場オペレーションの負荷が高い

と感じる場面が多く、  
「自分で仕組みを作れば改善できるのでは」と考え、個人開発しました。

## 使用技術
### フロントエンド
- React
- TypeScript
- Vite

### バックエンド / BaaS
- Supabase（データ管理・認証）

### インフラ
- Netlify（デプロイ）

### その他
- Git / GitHub

## 機能概要
- 注文データの登録・管理
- 状態管理を用いたリアルタイムUI更新
- フロントエンドとバックエンドのデータ連携

## 工夫した点
- コンポーネント分割を意識した設計
- TypeScriptによる型安全性の確保
- 実際の店舗利用を想定した画面構成

## 苦労した点
- 開発途中でReactのバージョン違いによるエラーが発生
- エラーメッセージ・公式ドキュメント・Issueを調査し、
  原因を特定して解決
  
## 画面構成
- Login：ゲストログイン（ダミー）
- Select：役割選択（ホール / キッチン / カウンター）
- Hall：注文入力・送信
- Kitchen：調理ステータス管理
- Counter：配膳管理
